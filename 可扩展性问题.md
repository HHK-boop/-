# Week 3 可扩展性问题总结

## 1. 可直接复用的部分

- 字段结构：Week2的company_list字段可以直接迁移到2024、2023。
- PDF来源：北交所相关公告中的披露文件链接在历史年份中仍然可用，但直接用urllib访问可能被重定向或403，需要模拟浏览器下载。
- 文件版本规则：标题包含“招股说明书”或“向不特定合格投资者公开发行股票并在北京证券交易所上市招股说明书”的正式稿规则有效。
- 关键词定位：历史沿革、股本形成、股权转让、增资、私募基金、特殊投资条款等关键词仍能定位候选页。

## 2. 需要修改的部分

- 企业list构建逻辑需要支持多年份和抽样规则。
- 北交所官网股票列表只稳定返回证券简称，完整公司名称需要后续从招股书首页或概况页补全。
- 北交所PDF下载需要处理防盗链/重定向，不能完全沿用原有静态PDF批量下载方式。
- 日志必须按周单独命名，防止Week2和Week3相互覆盖。
- 抽取结果仍是候选级，不能直接作为最终PE/VC数据库。

## 3. 主要风险

- 部分招股书披露页码较靠后，固定解析前180页可能遗漏少数发行前股东表或特殊条款。
- 规则抽取容易把员工持股平台、历史股东、产业方、基金管理人误判为PE/VC投资方。
- 老师提供的友升股份三表示范说明，最终结果应拆成认缴流量和股权结构存量两张事实表，当前候选JSON仍需人工深度整理。

## 4. 改进方向

- 后续用目录识别替代固定前180页策略。
- 引入三表结构：subscription_flow、equity_snapshot、schema_cross_check。
- 对每家公司抽样核查总股本闭合、逐股东变化和来源页码。
